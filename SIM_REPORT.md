# Suitspire 模拟器诊断报告

**样本：1000 局 AI 自动模拟（src/simulator.ts，运行 `npx tsx src/simulator.ts --runs=1000`）**

AI 策略：每局开局随机选一个目标流派（intendedBuild），起手特性 / 战利品 / 商店购买都偏好该流派；战斗中按"装备 → HP 救命 → 大招 → buff → 攻击 → 花色操作 → 其他"优先级出牌；铁匠铺只附目标流派的附魔；地图节点偏好 forge > shop > event > elite > battle。

---

## 1. 整体难度（核心）

| 指标 | 值 |
|---|---|
| 平均通关层 | **4.16** |
| 中位 | 3 |
| P25 / P75 | 3 / 5 |
| 最远到达 | 12（10 局，1%）|
| HP=0 死亡 | 958 / 1000（96%）|
| 卡死/超时 | 32（3.2%）|

**核心结论：F3 是巨型难度墙。** 626/1000 (62.6%) 玩家死在 F3，整体死亡分布严重头重脚轻。

```
F 3:  626 █████████████████████████   ← Boss 关，62.6% 死在这
F 4:   61 ██
F 5:   94 ████
F 6:  136 █████                         ← 第 2 个 Boss 关
F 7:   16 █
F 8:   17 █
F 9:   23 █
F10:    8
F11:    4
F12:   14 █                              ← 1% 通顶
```

第 1-2 关玩家几乎不会死（牌库够薄、敌人弱）→ **难度从 F2 → F3 跳跃太陡**，玩家还没成型 build 就撞 Boss。

## 2. 流派失衡

| 流派 | 玩家数 | 平均关 | T1+ | T2+ | T3 | 大招总数 |
|---|---|---|---|---|---|---|
| **♥ 红心** | 234 | **5.1** | 95% | **40%** | **3%** | **1521** |
| ♠ 黑桃 | 271 | 4.1 | 98% | 36% | 1% | 116 |
| ♣ 梅花 | 251 | 4.0 | 96% | 26% | 0% | 174 |
| ♦ 方块 | 244 | **3.5** | 96% | 28% | 0% | 77 |

**问题**：

1. **♥ 红心吸血流过强**：平均关数比其他流派高 **1-1.6 关**，T3 触发率 3% 是其他 6 倍，大招总数 1521 是其他流派总和（367）的 4 倍。原因：T3 大招"**生命洪流**（HP 回满 + maxHP +5）"让玩家进入"打掉 Boss → 大招回满 → 再打"循环，几乎可以无限续航。
2. **♦ 方块灵巧流最弱**：平均关数 3.5 几乎没过 F3，T3 触发率 0%。原因：闪避 build 数值上没达到一锤定音的程度（cap 75% 闪避并非永不受击），缺少类似"生命洪流"的高威力大招。
3. **♠ ♣ 中等**：T2 触发率 26-36%，能感受到 Tier 1 效果但很难凑到 T3。

| 排名 | 卡 | 总次数 | 平均/局 |
|---|---|---|---|
| 1 | **回血药水** | 25000 | 25.0 |
| 2 | 磨刀石 | 17310 | 17.3 |
| 3 | 撕裂 | 16734 | 16.7 |
| 4 | 攻击 ♠ | 15654 | 15.7 |
| 5 | 强化药 | 14876 | 14.9 |
| 6 | 攻击 ♥ | 14106 | 14.1 |
| 7 | 聚气 | 13945 | 13.9 |
| 8 | 攻击 ♣ | 12850 | 12.9 |
| 9 | 攻击 ♦ | 12800 | 12.8 |

**洞察**：
- **没有"从没被打出的卡"**——所有牌都至少被出过（健康）
- 回血药水 25/run 显著超过任何其他单卡——表明玩家**严重依赖回血保命**而非靠 build 输出
- 攻击牌 4 色使用率均衡（12-15/run）→ 花色相性机制工作正常
- **撕裂 16.7/run 偏高异常**：AI 把它当通用 buff 出，但实际对低 floor 无 armor 敌人无意义——说明撕裂的"无 armor 时无效"的反馈不够明显，新手玩家会浪费手牌

## 4. 装备 / 附魔获取

### 武器持有率（玩家终局装备的武器）

| 武器 | 持有 | 占比 | 备注 |
|---|---|---|---|
| 匕首 | 386 | 38.6% | 起始基础（♥） |
| 短剑 | 232 | 23.2% | 起始过渡 |
| 战弓 | 161 | 16.1% | 起始基础（♦） |
| 长剑 | 146 | 14.6% | 起始基础（♣） |
| 巨锤 | 56 | 5.6% | rare build |
| 法杖 / 链刃 | 53-55 | 5.5% | rare build |
| 法师杖 / 双刀 | 47-48 | 4.8% | super_rare |
| 血裂刃 | 35 | 3.5% | rare ♥ |
| 破军 / 狂剑 / 连弩 | 13-23 | 1-2% | super_rare |
| **divine_blade / excalibur** | **2 / 2** | **0.2%** | **史诗（极罕见）** |

**结论**：玩家几乎全程靠基础装备，**build 武器命中率 5-15%，史诗武器 < 1%**。这导致：
- 高级武器机制（如 wizard_staff 的"非攻击牌累积"、berserker_blade、repeating_bow 等）**几乎没人触发**
- 起始牌库的过渡武器主导整局——玩家**进入不到深度 build**

### 附魔分布

铁匠铺访问 946 次，成功附魔 382 次（**40% 转化率**）—— 60% 玩家去了铁匠铺但碎片不够。

| 附魔 | 成功 | 占比 |
|---|---|---|
| ec_phalanx 重甲列阵 | 82 | 8.2% |
| e_strategist 算计 | 72 | 7.2% |
| e_brawler 强袭 | 35 | 3.5% |
| ec_resilient 守护契 | 28 | 2.8% |
| ec_lifesteal 血祭仪 | 22 | 2.2% |
| ec_focus 凝神 | 15 | 1.5% |
| e_reaper 收割 | 7 | 0.7% |
| ec_warblood / ec_runic | 4 | 0.4% |
| ec_arcane / 其他 | 1-3 | ~0% |
| **ec_swift（双稀少究极）** | **0** | **从没人附过** |

**结论**：
- 13 个附魔中只有 **2 个普通附魔 + 2 个复合附魔** 出现率 > 2%
- **究极附魔 ec_swift（双稀少）** 在 1000 局里 0 触发，完全是"死内容"
- 复合附魔配方对碎片要求过高 → 玩家在能附的时候就附了普通的，凑不齐复合

## 5. 商店 / 碎片经济

商店访问 948 次，购买 322 张（**0.34 张/访问**，平均每次只买 0.3 张）。

**商店购买 Top 10**：

| 卡 | 次数 | 类型 |
|---|---|---|
| 变色 | 42 | 改色 |
| 皮甲 | 31 | 基础装备 |
| 长剑 | 26 | 基础装备 |
| 攻击 ♥ | 24 | 攻击牌 |
| 匕首 | 22 | 基础装备 |
| 共鸣咒 | 17 | 改色 |
| 战弓 | 15 | 基础装备 |
| 染色术 | 15 | 改色 |
| 圆盾 | 14 | 基础装备 |
| 重甲 | 14 | 基础装备 |

**结论**：
- 玩家在商店**只买基础装备 + 改色卡**——商店货架上 super_rare/epic 卡没人买
- 0.34 张/访问 = 玩家碎片不够、不愿花碎片在商店上 → **商店物价偏贵**
- super_rare = 10 碎片，epic = 30 碎片 —— 玩家通常 F3 死亡时碎片只有 5-10，根本买不起 super_rare/epic

## 6. 异常诊断

**41 局有错误（4.1%）**：
- 32 局 stuck（AI 在战斗 phase 卡 30+ 次外层 loop）
- 9 局 force-quit（战斗超 50 回合还没结束）

主要分布在 F3-F10，少数 F4 / F6 也卡——这些是 build 没成型时撞较强敌人造成的 stalemate（伤害不够 + 没死也死不了）。AI 不会主动"放弃"，但人类玩家也会面临类似困境（"打不动但死不了"）。

## 7. 关键问题与建议

### 🔴 严重问题（必须修）

**P1：F3 死亡墙过陡（62.6%）**
- 现状：F1-F2 几乎不死 → F3 Boss 直接 62% 团灭
- 原因：F3 玩家通常 0-1 件 build 装备 + 0 附魔，对上 +3.2× HP / +1.30× atk 的 Boss
- 建议：
  - **F3 Boss HP 从 ×2.60 → ×2.20**（已经从 ×3.2 → ×2.6，但仍然偏硬）
  - 或：F1-F2 增加普通战难度作为预热，平摊学习曲线
  - 或：第一次 Boss 改为 elite（去掉皇冠 buff），让 F6 才是首个 boss
  - 或：F2 强制送 1 件 build 装备/附魔

**P2：流派失衡 — ♥ 过强 / ♦ 过弱**
- ♥ T3 "生命洪流"循环再生太爆 → 单局触发 100+ 次
- 建议：
  - "生命洪流"：HP 回满 + maxHP +5 改为 **HP 回 50% + maxHP +3**，并且 **整局限 3 次**
  - ♦ 灵巧流补一个"决定性"T3 大招效果（当前是闪+三连击；可加 "敌人下回合行动被取消" 之类的强制控制）
  - 或：拉高 ♦ T2 阈值收益（每攻击 +1 hit 概率从 25% → 35%）

**P3：高级装备几乎接触不到**
- super_rare 武器持有率 < 5%，史诗 < 0.2%
- 建议：
  - 提高 super_rare/epic 出现率（floor 6+ 时 super_rare 权重再 +50%）
  - 或：每 3 关（F3 boss 通关后、F6 通关后、F9 通关后）强制送 1 张 super_rare 武器/防具供玩家选

**P4：附魔系统利用率低（40% 转化率）**
- 60% 玩家去了铁匠铺白跑
- 复合附魔触发率 < 5%，究极 ec_swift = 0
- 建议：
  - **铁匠铺访问时检测碎片是否够**——不够就允许"碎片折扣"或"延迟附魔"
  - 或：附魔配方稍微降配（普通从 ×3 → ×2，复合从 ×2+×2 → ×2+×1）
  - 或：让玩家通过商店主动卖碎片（已有 sell card）+ 提高卖卡价让 carry 更多碎片

**P5：商店物价偏高**
- 0.34 购买/访问，rare 7 碎片 / SR 10 / Epic 30 玩家买不起
- 建议：
  - rare: 7 → 5
  - super_rare: 10 → 8
  - epic: 30 → 18
  - 或：每 N 关给玩家一笔"碎片大礼包"（5-10 碎片任选种族）

### 🟡 中等问题（建议改）

**P6：撕裂被滥用（16.7/run）**
- AI/新手会无脑出，但低关无 armor 敌人浪费手牌
- 建议：撕裂卡描述加红字提示"对无护甲敌人无效"，或对无 armor 敌人直接报错不消耗

**P7：玩家依赖回血药水（25/run）**
- 每局打 25 次回血——build 在堆数值的同时玩家其实在"龟"
- 建议：调低 it_heal 在起始牌库的数量（2 → 1），或回血量降低（最少 12 → 10），让玩家更依赖 build 输出

**P8：4.1% 异常率（stuck / force-quit）**
- 主要是 build 不行时打不动也死不了
- 建议：每关进入 floor_map 时检查"玩家是否有可叠加输出"——可能 design-level 不需要解决，但需要给玩家"主动放弃"的入口（撤退到下一关减 HP？）

### 🟢 健康项

- ✅ 所有卡都至少被打出（无死牌）
- ✅ 4 色攻击牌使用率均衡（12-15/run），花色相性运作正常
- ✅ 起始装备覆盖 4 花色，玩家能基础上手
- ✅ 战斗机制无明显 bug（除了 stuck 是 AI 死循环而非游戏 bug）

## 8. 建议优先级 / 实施顺序

| 优先级 | 改动 | 难度 | 预期效果 |
|---|---|---|---|
| **P1.1** | F3 Boss HP ×2.60 → ×2.20 | 1 行代码 | F3 死亡率 62% → 预期 50% |
| **P2.1** | 生命洪流改 +HP 50% / maxHP +3 / 整局 3 次 | 5 行代码 | ♥ 平均关 5.1 → 预期 4.5（拉平） |

## 9. simulator.ts 用法（已推到 dev）

```bash
# 跑 1000 局，输出报告到 stdout，原始 JSON 到 /tmp/results.json
npx tsx src/simulator.ts --runs=1000 --maxFloor=12 --out=/tmp/results.json

# 单局调试
npx tsx src/simulator.ts --runs=1 --maxFloor=6 --verbose
```

AI 策略源代码在 `src/simulator.ts`，可以调整：
- `pickIntendedBuild()` — 玩家流派偏好分布
- `aiPlayBattle()` — 战斗出牌优先级
- `aiPickRewardCard()` — 战利品选择评分
- `aiForge()` — 铁匠铺策略

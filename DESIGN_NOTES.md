## 🅒 闪避 / 穿甲系统补完（待实现）

### 当前问题
1. **闪避：**
   - 意念甲（10/20/30/40% 完全闪避）和特性 p_dodge（每张 +3% 完全闪避）独立 roll，玩家算不清最终概率。
   - "闪避姿态"（技能：受击减半）和"完全闪避"是两个完全不同的概念，名字撞了。
   - 战斗 UI 不显示当前闪避概率。
   - 触发时只有日志一行，毫无爽感。

2. **穿甲：**
   - 全游戏只有长剑一件破甲（pierce: 3）。
   - 后期敌人 armor 增长到 4-5，破甲 3 形同虚设。
   - 没有破甲 buff、技能、附魔——破甲是个一锤子买卖，没有 build 空间。

### 实施方案

**闪避（统一术语）：**
- 把闪避数值统一成单一字段：`player.statuses` 里加 `dodge_chance` 状态，stacks = 当前百分比
- 所有来源（意念甲、p_dodge、新技能）都通过 `addStatus` 累加到同一个 stacks
- battle.ts 里只在一处 roll：`if (Math.random() < dodgeChance) skip damage`
- "闪避姿态" 技能改名为 "**屏息**"，避免和闪避概念撞名
- 战斗 UI：玩家区显示"🎯 闪避 35%"芯片
- 触发动画：半屏闪烁 + 中心 "MISS" 浮字 + 敌人 0.2s 摇晃
- 新卡：
  - 道具 **烟雾弹**（rare）：本场战斗 +30% 闪避，5 回合
  - 技能 **风步**（super_rare）：下一次受击必闪避（一次性 buff）
  - 附魔 **幻影附魔**（消耗 dark 碎片 ×3）：闪避后下次攻击双倍伤害

**穿甲（build 化）：**
- 给 pierce 加 build 维度，新增：
  - 武器 **破军**（rare，奖励池）：基础伤 5，pierce 等于敌人 armor 的 50%（自适应）
  - 技能 **穿甲射**（rare）：无视全部 armor 一击
  - 附魔 **锐利**（消耗 humanoid 碎片 ×3）：所有攻击 +pierce 等于楼层数
  - 特性 **洞察**：每张 pierce +1
- 攻击 log 里破甲量直接打在伤害飘字上（如 "-15 (破甲 3)"）

### 涉及文件
- `types.ts`（新状态字段）
- `battle.ts`（统一 roll、伤害飘字）
- `cards.ts`（新卡、新附魔、新特性）
- `main.ts`（闪避概率芯片、MISS 动画）
- `style.css`（动画 CSS）

## 🅓 楼层间随机事件（待实现）

### 目标
在战斗外加一层"决策"，做出每次跑酷的差异化——肉鸽核心循环。

### 触发时机
每 2 关之间（即将进入 3/5/7… 关时），目前是铁匠铺位置。
**改为：先随机事件，再铁匠铺。** 或者把铁匠铺也并入事件池作为其中一个事件。

### 事件池（6-8 个）

| 事件 | 描述 | 玩家选择 |
|---|---|---|
| 🛒 **流浪商人** | 任选 5 张（来自奖励池）展示 | 选 1 张，消耗对应稀有度的灵魂碎片：common 3、rare 5、epic 10 |
| ⛲ **古老神龛** | 献祭 5 HP 永久换 1 张稀有牌 | 接受 / 拒绝 |
| 💀 **诡异棺木** | 50% 1 件 epic / 50% 中诅咒（永久 -2 攻击） | 开 / 不开 |
| 🔥 **远行篝火** | 回 30% HP 或升级 1 张牌至 +形态（暂时未实现牌升级，可延后） | 二选一 |
| 🎲 **赌徒** | 弃 2 张换 1 张随机奖励池牌 | 接受 / 拒绝 |
| 🐦‍⬛ **流浪术士** | 免费选 1 个特性的小弱化版（stacks=0.5 效果） | 三选一 |
| 📦 **神秘宝箱** | 抽 1 张 epic（30% 概率有陷阱：HP -30%） | 开 / 不开 |
| ⚒ **铁匠铺**（保留作为事件之一） | 当前铁匠铺逻辑 | 现有 |

### 实施方案
- 新 GamePhase: `"floor_event"`
- 新 EventDef 接口：`{ id, name, icon, desc, options: EventOption[] }`
- `event.ts` 新文件，事件池 + 触发逻辑
- 在 `afterPerkReward` 里：每 2 关后改为 → 先 floor_event → 完成后再 forge → 再 startNextFloor
- main.ts 新增 `renderFloorEvent()`：图标 + 描述 + 选项按钮
- 选择后立刻应用效果，自动进入下一阶段
- 部分事件需要"花钱"（碎片消耗），UI 要显示当前碎片库存

### 涉及文件
- `types.ts`（GamePhase 加 floor_event）
- `event.ts`（新文件，事件定义与逻辑）
- `game.ts`（流程调度）
- `main.ts`（渲染）
- `style.css`（事件 UI）

### 注意
- 新增的"+ 形态升级"和"碎片消耗买卡"是新机制，第一版可以简化（没有 + 形态时改为加 1 张 rare）
- 事件之间互斥：每两关只触发一个，避免节奏拖沓
- 第 N+1 次重复跑酷时，不要重复出现"棺木陷阱"等惩罚事件——可以引入 hash(seed) 排重（高级，可延后）

## 中后盘内容 / 重玩驱动力（待评估）

> 玩家完成第一次跑酷（floor 6-9）后，需要"再来一次"的明确驱动力。
> 以下都是常规肉鸽思路，**用户反馈"有点太传统了"**，记下来等想到秘制酱料再回头看。

### 候选 1：Ascension（升华）系统
通关后解锁难度档（A1-A10），每档加一个 modifier：
- A1 敌人 HP +20%
- A2 起手少 1 张
- A3 普通带精英特能
- A4 Boss 招式 +2
- A5 神秘事件陷阱率翻倍
- A6 Epic 概率减半
- A7 每关末必触 enemy_first
- A8 玩家伤害 -10%
- A9 敌人攻击 +20%
- A10 终极考验 + 隐藏 Boss

工作量：~200 行（modifier 系统 + UI 显示 + 升档逻辑 + localStorage）

### 候选 2：天命卡 / Capstone
每个种族 25 个碎片解锁专属"天命卡"（永久解锁，今后所有跑酷的奖励池都会出）：
- 兽魂之牙（兽）：每场战斗第 3 回合开始 +50% 攻击直到结束
- 君王盾牌（人型）：每回合开始 +5 临时护甲
- 不死之书（不死）：弃牌堆非攻击牌 ≥5 时每回合多摸 1 张
- 巨怪图腾（巨怪）：HP <50% 所有伤害减 50%
- 影之锁链（暗影）：每杀 3 个敌人下次攻击 ×3

工作量：~150 行（5 张卡 + 解锁判断 + 收集进度条 UI + localStorage）

### 候选 3：关卡主题
每 3 关切主题，让中后期不是"同一战斗，更高数值"：
- 1-3 关：标准
- 4-6 关：寒霜（敌人自带冰冻 / 玩家行动慢一拍）
- 7-9 关：炼狱（战斗结束 -10% HP / 必带回血卡）
- 10+ 关：虚空（手牌随机变诅咒卡）

工作量：~250 行（主题系统 + 各主题机制 + UI 预告）
